# Быстрое решение проблемы ERR_CONNECTION_REFUSED

## Проблема
Браузер показывает "ERR_CONNECTION_REFUSED" при попытке открыть http://localhost:8000/ui

## Решение

### Вариант 1: Используйте 127.0.0.1 вместо localhost

В браузере откройте:
- **http://127.0.0.1:8000/ui** (вместо localhost)
- **http://127.0.0.1:8000/docs** (API документация)

### Вариант 2: Перезапустите сервер

Сервер уже перезапущен с правильной конфигурацией. Просто обновите страницу в браузере (F5 или Cmd+R).

### Вариант 3: Проверьте, что сервер запущен

```bash
# Проверьте статус
lsof -i:8000

# Если сервер не запущен, запустите:
source venv/bin/activate
python main.py
```

## Проверка работы

1. **Откройте в браузере:**
   - http://127.0.0.1:8000/ui
   - http://127.0.0.1:8000/docs

2. **Или проверьте через curl:**
   ```bash
   curl http://127.0.0.1:8000/api/health
   curl http://127.0.0.1:8000/ui
   ```

## Причина проблемы

Проблема была связана с тем, что браузер пытался подключиться через IPv6 (::1), а сервер слушал только на IPv4. Теперь сервер настроен на явное использование 127.0.0.1.
